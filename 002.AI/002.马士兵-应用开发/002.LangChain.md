# 概念

## Agent （智能体）
将语言模型与工具相结合，创建可以推理任务、决定使用哪些工具并==迭代==寻求解决方案的系统。

1. **模型调用**- 使用提示和可用工具调用 LLM，返回响应或执行工具的请求。
2. **工具执行**- 执行 LLM 请求的工具，返回工具结果。

这个循环一直持续到 LLM 决定完成为止

## Middleware（中间件）
中间件提供了一种更严格地控制 Agent 内部发生的事情的方法。中间件在每个步骤之前和之后都可以设置钩子。

- 监控：通过日志记录、分析和调试跟踪代理行为
- 修改：转换提示、工具选择和输出格式。
- 控制：添加重试、回退和提前终止逻辑。
- 其他：应用速率限制、护栏和 PII 检测

## Guardrails（护栏）
为您的代理实施安全检查和内容过滤。帮助您构建安全、合规的 AlI 应用程序。他们可以检测敏感信息、执行内容策略、验证输出并在不安全行为引起问题之前防止它们。

- 防止 PIl 泄漏（个人敏感信息拦截）
- 检测和阻止提示注入攻击（智能体安全问题）
- 屏蔽不当或有害内容
- 执行业务规则和合规性要求
- 验证输出质量和准确性

## 上下文工程
上下文工程是解决智能体的相应速度和提高智能体准确率的最重要的解决方案

LLM 失败的原因有以下两个：
1. 底层 LLM 能力不够；
2. “正确”的上下文没有传递给 LLM。

上下文工程是以正确的格式提供正确的信息和工具，以便 LLM 能够完成任务。这是人工智能工程师的头号工作。这种缺乏“正确”上下文是更可靠代理的头号障碍，而 LangChain 的代理抽象经过独特设计，以促进上下文工程。
