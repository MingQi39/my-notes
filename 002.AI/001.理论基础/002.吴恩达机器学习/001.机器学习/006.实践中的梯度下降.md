假设一个二元线性回归问题：特征为房子大小 $x_1$ 和卧室数 $x_2$，目标为房子价格 $y$

- 房子大小 $x_1$ 的取值范围为 300~2000（范围大）
- 卧室数 $x_2$ 的取值范围为 0~5（范围小）

一个训练数据为 $x_1 = 2000, x_2 = 5, y = 500$，找合适的 $w_1, w_2, b$ 进行拟合

1. $w_1 = 50, w_2 = 0.1, b = 50$

解得 $\hat{y} = 50 * 2000 + 0.1 * 5 + 50 = 100,050.5$，拟合效果非常差

2. $w_1 = 0.1, w_2 = 50, b = 50$

解得 $\hat{y} = 0.1 * 2000 + 50 * 5 + 50 = 500$，拟合效果好很多

所以：当一个特征的可能取值范围较大时，其参数的合理值会相对较小；反之当一个特征的可能取值范围较小时，其参数的合理值会相对较大

![[Pasted image 20250811125408.png]]

这是因为 $w_1$ 的微小变化会对预估价格产生很大影响，进而对成本 $J$ 产生很大影响；相比之下 $w_2$ 需要有更大的变化才能对预测产生同样大的影响

这意味着：运行梯度下降可能出现的结果，如果你直接照原样使用训练数据，由于等高线又高又狭长，梯度下降最终可能会来回振荡，反复多次最终才能找到全局最小值

# 特征缩放

特征缩放（Feature Scaling）能使梯度下降运行速度大幅提升

这意味着对你的训练数据进行一些变换，使得 $x_1$ 和 $x_2$ 的取值范围接近，比如都是 0~1 之间

![[Pasted image 20250811125936.png]]

回顾一下：当你有取值范围差异很大的不同特征时，这会使梯度下降运行得很慢；但对不同特征进行重新缩放让它们的值域范围相近，能大幅加快梯度下降

# 判断梯度下降是否收敛

梯度下降规则：

$$
\left \{
\begin{array}{c}
w_j = w_j - \alpha \frac{\partial}{\partial w} J(\vec{w}, b) \\
b = b - \alpha \frac{\partial}{\partial b} J(\vec{w}, b)
\end{array}
\right.
$$

目标是 $\underset{\vec{w}, b}{\operatorname{minimize}} J(\vec{w}, b)$

## 绘制学习曲线

![[Pasted image 20250811132258.png]]

若梯度下降正常运行，则每次迭代后成本 $J$ 应降低

若 $J$ 在一次迭代后增加，意味着学习率 $\alpha$ 的选择可能有误（通常为 $\alpha$ 值过大），或者代码存在问题

上图中，当达到约 300 次选代时，成本 $J$ 趋于稳定，不再大幅下降，曲线已趋平缓，这意味着梯度下降已经收敛

梯度下降收敛所需的选代次数在不同应用中差异很大，事实证明很难预先判断梯度下降需要多少次迭代才能收敛

所以需要绘制一条学习曲线进行观察，以试图找出何时可以停止训练模型

## 自动收敛测试

另一种判断模型训练是否完成的方法是进行自动收敛测试

令 $\epsilon = 10^{-3}$，如果在一次送代中代价函数 $J$ 的下降小于这个 $\epsilon$ 值，那么很可能位于上图学习曲线的平缓部分，这时就可以宣布收敛了

但是通常选择合适的阈值 $\epsilon$ 很难，所以实际上更倾向于看学习曲线图，而不是依靠自动收敛测试判断梯度下降是否收敛

# 选择合适的学习率

若学习率太小，梯度下降算法运行的很慢；若学习率太大，梯度下降算法可能无法收敛（[[004.梯度下降#学习率|详见]]）

如何为你的模型选择一个合适的学习率？

![[Pasted image 20250811134059.png]]

绘制学习曲线观察到如上图两种情况，可能是因为学习率太大或代码有问题

正确实现梯度下降的一个调试技巧是：使用足够小的学习率，代价函数应在每次选代时减小

即使将 $\alpha$ 设为非常小的数，每次选代时 $J$ 都不减小，反而有时会增大，通常意味着代码中有错误

但将学习率设置太小，梯度下降算法运行的很慢。所以通常会尝试一系列学习率的值：

0.001 → 0.003 → 0.01 → 0.03 → 0.1 → ...（每次约增大三倍）

直到找到一个太小的值和一个太大的值，然后慢慢尝试选择尽可能大的学习率或者只是比找到的最大合理值略小的值，通常是一个很好的学习率
