本文档面向需要接入 SNP2（Science Navigator Protocol 2）协议的 Agent 开发者，主要说明 `msg.channel.uiInfo` 中各字段的含义和使用方式，以及 `actionList` 的处理细节

**注：本指南主要介绍与前端通信和** **UI** **渲染部分**

# 消息结构概览

SNP 协议的核心是 `ChatMessage`，每条消息包含两个主要部分：

```ts
export interface ChatMessage {
    sessionId: string;
    system?: MessageSystem;
    channel: {
        schema?: ChannelSchema;
        version?: string;
        agent?: string;
        branchId?: string;
        sourceSessionId?: string;
        sourceQuestionId?: string;
        questionId: string;
        answerId: string;
        messageId: string;
        role: Role;
        auth: AuthStatus;
        uiInfo: UIInfo;
        entities?: any[];
        state?: ChannelState;
        meta?: ChannelMeta;
    };
}
```

- **system**：系统级事件（心跳、开始、结束、错误等）
- **channel**：前端渲染相关的所有信息，**Agent 关注点在这里**
- 其他还有 `a2aInfo`, `agentInfo`, `mcpInfo` 等字段， 本指南不拓展讲解

# `channel.uiInfo` 字段详解

`uiInfo` 是 Agent 输出给前端渲染的核心载体：

```ts
export interface UIInfo {
    localMessageId?: string;
    layout: ChatMessageContributions;
    type: MessageType;
    subType: string;
    content: { status?: MessageStatus } & DataMessageContent & Record<string, any>;
    response?: Record<string, any>;
    siderId?: string;
    actionList?: { key: string; action: ActionType }[];
}
```

## `type` 消息类型

```ts
export type MessageType = 'ui' | 'data' | 'error' | 'custom';
```

| 值        | 用途                                                |
| -------- | ------------------------------------------------- |
| `ui`     | **最常用**，表示这条消息需要渲染到界面上                            |
| `data`   | 纯数据消息，不渲染，但会写入前端数据存储（session/answer 级别的 DataTree） |
| `error`  | 错误消息                                              |
| `custom` | 自定义类型，留给特殊场景                                      |

**判断方式**：

```ts
export const uiMessage = (msg: ChatMessage): boolean => {
	return feMessage(msg) && msg.channel?.uiInfo?.type === 'ui';
};

export const dataMessage = (msg: ChatMessage): boolean => {
	return feMessage(msg) && msg.channel?.uiInfo?.type === 'data';
};
```

## `layout` 渲染位置

决定这条消息渲染到聊天窗口的哪个区域：

```ts
export type ChatMessageContributions = 'preprocessor' | 'main' | 'postprocessor' | 'sider' | 'actions';
```

| 值               | 位置说明               |
| --------------- | ------------------ |
| `preprocessor`  | 回答区域**上方**的预处理内容   |
| `main`          | 回答区域的**主体内容**（最常用） |
| `postprocessor` | 回答区域**下方**的后处理内容   |
| `sider`         | 右侧边栏内容             |
| `actions`       | 操作按钮区域             |

## `subType` 渲染器类型

指定使用哪个插件/组件来渲染这条消息。格式通常是：`@namespace/plugin-name`

例如：

- `@bohrium-chat/common/markdown` — 通用 Markdown 渲染
- `@bohrium-chat/common/sider-tab` — 侧边栏 Tab 组件

前端会根据 `subType` 查找对应的注册组件进行渲染

## `content` 消息内容

这是一个**任意类型的对象**，包含实际要渲染的数据。结构由 `subType` 对应的插件约定

保留字段（约定）：以下字段不能被业务字段覆盖和定义

| 字段       | 类型              | 说明                    |
| -------- | --------------- | --------------------- |
| `status` | `0` \|`1` \|`2` | 等待、成功、失败              |
| `text`   | `string`        | 常用于 Markdown 类消息的文本内容 |
| `data`   | `any`           | `type=data` 时的数据载体    |
| `route`  | `DataRoute`     | 局部更新时指定数据写入路径         |

## `siderId` 侧边栏关联 ID

当 `layout='sider'` 时，用于标识这条消息对应哪个侧边栏实例

## `localMessageId` 本地消息 ID

前端生成的临时唯一 ID，用于消息去重和本地状态追踪

# `actionList` 处理详解

`actionList` 是 SNP 协议的核心特性之一，用于描述**如何更新已有消息的内容**，实现流式输出、增量更新等效果

当前 actionList 是 SNP 自主实现的 data patch 操作

后续计划使用标准的 JSON Patch 代替

## 结构

```ts
actionList?: { key: string; action: ActionType }[];
```

每个 action 包含：

- `key`: 要更新 `content` 中的哪个字段
- `action`: 更新策略

## `action` 类型

```ts
export type ActionType = 'append' | 'delta' | 'replace' | 'merge';
```

| action    | 行为说明                                                                                                             |
| --------- | ---------------------------------------------------------------------------------------------------------------- |
| `append`  | 新增一条消息 （第一条消息必须是 append, 否则不会入栈）                                                                                 |
| `replace` | 直接替换：`content[key] = newValue`                                                                                   |
| `delta`   | 增量追加：<br><br>- 字符串：拼接 `oldVal + newVal`<br>- 数组：合并 `[...oldVal, ...newVal]`<br>- 对象：浅合并 `{...oldVal, ...newVal}` |
| `merge`   | 对象浅合并                                                                                                            |

## 处理逻辑源码

```ts
export const delta = (params: ActionParams) => {
	const { key, oldContent, content } = params;

	const oldVal = oldContent[key];
	const newVal = content[key];
	const newContent = { ...oldContent };
	if (typeof oldVal === 'string' && typeof newVal === 'string') {
		newContent[key] = oldVal + newVal;
	} else if (Array.isArray(oldVal) && Array.isArray(newVal)) {
		newContent[key] = [...oldVal, ...newVal];
	} else if (oldVal && typeof oldVal === 'object' && newVal && typeof newVal === 'object') {
		newContent[key] = { ...oldVal, ...newVal };
	} else {
		newContent[key] = newVal;
	}

	return newContent;
};
```

# `type=data` 消息处理

`type=data` 的消息不会渲染到界面，而是写入前端的数据存储，供其他组件读取

## 数据路由

```JavaScript
export type DataScope = 'session' | 'answer';

export interface DataRoute {
    data?: DataScope;
    /** optional sub-path for targeted updates like 'pluginA.metrics' */
    path?: string;
}
```

| 字段           | 说明                                  |
| ------------ | ----------------------------------- |
| `route.data` | 数据作用域：`session`（会话级）或 `answer`（回答级） |
| `route.path` | 嵌套路径，如 `pluginA.metrics`，支持点号分隔     |

## 处理流程

```ts
export function applyDataMessage(msg: ChatMessage) {
    if (!Is.dataMessage(msg)) return;

    const uiInfo = Parse.getUiInfo(msg);
    const pluginId = uiInfo?.subType || '';
    const answerId = Parse.resolveFEMessage(msg)?.answerId || msg.channel?.answerId || '';
    const actionList = uiInfo?.actionList || [];
    const incoming = (uiInfo?.content || {}) as DataMessageContent;
    const trees = useDataTreeStore.getState();
}
```

- 默认 action 为 `merge`
- 数据会写入 `DataTreeStore`
- 其他组件可以通过 `useDataTreeStore` 订阅数据变化

```json
[
  {
    "channel": {

      "uiInfo": {

        "content": {

            "data": "answer",

  {

      "answerId": "abc-123",
```

# 总结

- **`type=ui`**：需要渲染的消息，配合 `layout` 决定位置，`subType` 决定渲染器
- **`type=data`**：纯数据消息，写入前端存储供订阅使用
- **`actionList`**：控制增量更新策略，支持流式输出场景
- **常用 action**：`append`（流式文本）、`replace`（完全替换）、`merge`（对象合并）
